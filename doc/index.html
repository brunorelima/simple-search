SimpleSearch

Biblioteca javascript para pesquisas no estilo autocomplete com ou sem formulário e tabelas, podendo ser personalizado de acordo com as configurações passadas.

Exemplos

Conheça vários modelos de uso, utilizando formulário, tabelas ou paginação.

Ver detalhes »

Documentação

Consulte como funciona cada recurso da api, para configurar da melhor maneira.

Ver detalhes »

Exemplo de uso 
	
					
	var ssEstados =  new SimpleSearch({
		url: "/arquivos/controlador/consultarCmUf.php",
		query: "#inputNome",
		queryId: "#inputId",
		field: "ds_nome",
		fieldId: "id",  
		fieldRecords: "obj.registros", 
	});		
	
	<!-- Código HTML básico -->
	<input type="text" id="inputNome" />
	<input type="hidden" id="inputId" />
Importando arquivos

Biblioteca Javascript:

<script src="/recursos/simpleSearch/js/simpleSearch.min.js" type="application/javascript"> </script>


Caso o projeto não utilize o css do Twitter Bootstrap, é necessário importar um estilo css. Podendo escolher entre o css com estilo baseado no bootstrap ou baseado em telas antigas.


Estilo igual ao Twitter Bootstrap

<link rel="stylesheet" href="/recursos/simpleSearch/css/simpleSearch.min.css" type="text/css" />

Estilo baseado em tela de sistema legado

<link rel="stylesheet" href="/recursos/simpleSearch/css/simpleSearch.basic.min.css" type="text/css" />

Documentação

Atributos
Propriedade Descrição Exemplo
url Recebe a url que vai fazer a solicitação. url: "/arquivos/controlador/consultarCmUf.php"
query Seleciona o campo alvo para gerar a pesquisa query: "#inputTextName"
queryId Seleciona o campo que vai receber o valor selecionado da pesquisa queryId: "#inputValue"
field Informa o nome da propriedade da linha do sql que vai exibir nas linhas da pesquisa field: "ds_nome"
fieldId Informa o nome da propriedade da linha do sql que vai usar para popular o valor do 'queryId' fieldId: "id"
fieldRecords Informa o caminho o objeto json retornado que contém os registros fieldRecords: "obj.registros"
fieldSizePages Informa o caminho da propriedade que informa a quantidade de páginas totais existentes fieldSizePages: "obj.propriedades.qtdPaginasTotal"
minLength Informa a quantidade mínima de caracteres para realizar uma busca minLength: 3
paramSearch Nome do parametro padrão para pesquisas (Valor padrão: "busca") paramSearch: busca
templateField Faz um template com o nome dos atributos para aparecer na lista de itens templateField: "#ds_sigla# - #ds_nome#"
templateComplement Faz um template com o nome dos atributos para aparecer na lista de itens em baixo do campo selecionado templateComplement: "#ds_sigla# - #ds_nome#"
defaultValue Informa os valores dos registros para iniciar marcados defaultValue: [11, "Minas Gerais"] ou defaultValue: [[19, "Rio de Janeiro"], [8, "Espírito Santo"]]
defaultEmptyMsg Altera a mensagem padrão exibida quando não é retornado nenhum registro após a pesquisa. defaultEmptyMsg: "Nenhum registro para ser exibido."
data Informa na função de callback os parâmetros enviados antes da pesquisa data: function(){ return { filtro: "palavraFiltrada" }}
queryForm Informa o nome do formulário que vai serializado para passar os parâmetros para a busca queryForm: "#formEstados"
queryButton Informa a referência do botão do seletor que vai ser ativado as pesquisas queryButton: "#btnBuscaEstados"
queryContent Informa o container que vai receber o conteúdo da busca queryContent: "#containerExterno"
fieldPages Informa o caminho da propriedade do resultado que contém a lista de páginas exibidas na paginação. fieldPages: "obj.navegacao.paginas"
inputNames Informa o name dos inputs que estão sendo escolhidos para enviar como paramêtro na requisição inputNames: "estadosSelecionados[]"
delaySearch Informa quanto tempo esperar para poder realizar a próxima pesquisa (para evitar muita pesquisa em pouco tempo). (Tempo em milisegundos) delaySearch: 200
delayAutoSearch Informa quanto tempo para pesquisar automaticamente. (Tempo em milisegundos) delayAutoSearch: 2500
tableTitles Informa o título das colunas que vão ser formadas na tabela tableTitles: ["Sigla", "Nome"]
tableFields Informa o nome dos campos do objeto que vão ser exibidos pra preencher a tabela tableFields: ["ds_sigla", "ds_nome"]
tableFieldsTooltips Informa a legenda para ser exibida ao passar o mouse em cima dos fields tableFieldsTooltips: ["ds_sigla", "ds_nome"]
tableShowSelect Exibe a coluna de selecionar no ínicio da tabela tableShowSelect: true
tableKeepOpen Mantem a pesquisa aberta, mesmo após perder o foco. tableKeepOpen: true
tableShowClose Mostra botão para fechar a tabela. tableShowClose: true
tableLastColumn Adiciona o conteúdo na última coluna. tableLastColumn: "<button> #ds_nome# </button>", ou
tableLastColumn: function(){ return "<button> #ds_nome# </button>"; },
debug Ativa o modo debug, para exibir as mensagens com informações adicionais no console. debug: true
whenBlurClear Limpa o conteúdo caso não selecione algo e perca o foco. whenBlurClear: true
whenNullSetEmpty Quando vier "null" na resposta, limpa e mostra vazio, quando tiver atribuído a propriedade para "true". whenNullSetEmpty: true
whenSelectKeepOpen Quando seleciona o item, mantem a caixa aberta (usado com múltiplos itens). whenSelectKeepOpen: true
response Informa o retorno do objeto json, para não precisar fazer a requisição ajax. response: function( pagina, termoPesquisado ){ return pesquisaOffline( pagina, termoPesquisado ); },
responseChange Usado para manipular a resposta do servidor antes de popular o componente responseChange: function( response ){ return response; }
onselect Cria uma função para ser executado ao selecionar algum item da lista onselect: function( row ){ $("#destinoValor").val("Nome: " + row.ds_nome); }
onreset Cria uma função para ser executado ao resetar o componente onreset: function( row ){ $("#destinoValor").val(""); }
onsuccess Cria uma função para ser executado ao receber a resposta do servidor onsuccess: function( response ){ if (response.status == 'erro') throw Error("Deu erro na resposta do servidor!"); }
onerror Cria uma função para ser executado quando a resposta do servidor vem com algum tipo de erro onerror: function( response ){ util.alertaPadraoMsgErroAjax( response ); }
oncomplete Cria uma função para ser executado após tratar a resposta do servidor e ter montado o componente oncomplete: function( response ){ console.log("Manipulação de colunas por exemplo..."); }
onremoveselecteditem Executa ação ao clicar para remover um item selecionado da lista onremoveselecteditem: function(itemId){ alert(itemId); }
Funções
Nome Descrição Exemplo
disabled() Desativa a pesquisa do item ssEstados.disabled();
enabled() Reativa a pesquisa do item ssEstados.enabled();
select(id, nome, complemento)
'complemento': opcional 		Seleciona (ou adiciona) o item com o nome e id informado. ssEstados.select(19, "Rio de Janeiro");
ssEstados.select(19, "Rio de Janeiro", "RJ");
reset() Reseta o complemento para seu estado inicial. ssEstados.reset();
focus() Seta foco para o elemento. ssEstados.focus();
search() Executa a pesquisa, podendo passar como parâmetro o número da página também. ssEstados.search();
ssEstados.search(1);
getContainer() Retorna o id do componente criado. Exemplo: "#containerSimpleSearch5" ssEstados.getContainer();
disableRowByIndex( index ) Desabilita o select da linha informada. ssEstados.disableRowByIndex( 3 );
setMessage( msg ) Define a mensagem exibida abaixo do elemento, caso necessário exibir mensagem diferente da mensagem padrão. ssEstados.setMessage( "Não foram encontrados registros correspondentes às suas permissões de acesso." );
